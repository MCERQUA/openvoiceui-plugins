Before You Start

This wizard connects OpenVoiceUI to a Twenty CRM instance. Make sure the following are in place before continuing.

Requirements
  • Twenty CRM server running — self-hosted or cloud. The REST API must be accessible over HTTPS.
  • Domain with SSL — e.g. crm.yourdomain.com. Required for secure API calls and iframe embedding.
  • Wildcard subdomain (multi-tenant) — if using multi-workspace mode, you need *.crm.yourdomain.com DNS + a wildcard SSL certificate (Let's Encrypt DNS-01 recommended).
  • Nginx iframe headers — your CRM's nginx config must include: Content-Security-Policy: frame-ancestors https://*.yourdomain.com to allow iframe embedding.
  • Cloudflare users — if using Cloudflare, set the CRM subdomain to DNS-only (gray cloud). Cloudflare free plan does not issue certs for nested wildcards like *.crm.yourdomain.com.
  • No caching — set Cache-Control: no-store, no-cache on the CRM proxy. Stale iframes cause auth problems.

Workspace & API Key

You need a workspace on the CRM server and an API key for your agent to access it.

Create your workspace (if you don't have one)
  1. Open your CRM server in a new tab
  2. Click Sign up and create an account
  3. Complete the onboarding wizard to initialize your workspace
  4. Note your workspace subdomain from the URL bar
Generate an API key
  1. Log in to your CRM workspace
  2. Go to Settings (gear icon, bottom-left)
  3. Click APIs & Webhooks in the sidebar
  4. Click Create API key
  5. Set name: OpenVoiceUI Agent
  6. Set expiration: Never (or your preferred duration)
  7. Copy the generated token
Paste the full JWT token from Twenty CRM. This is stored securely on the server, never in the browser.
